向 Lustre 系统新增一个 OST 的操作步骤包括以下关键命令和过程：1. 在文件系统第一次进行格式化时，使用 mkfs.lustre 命令添加新的 OST，每个新的 OST 必须有一个唯一的索引，可使用 lctl dl 查看所有 OST 的列表。例如，添加一个新的 OST 至 testfs 文件系统，索引为 12：oss# mkfs.lustre --fsname=testfs --mgsnode=mdsl6 [KG] unknown_source。2. 如果 OST 永久不可用，可使用 lfs rm_entry {directory} 删除该 MDT 记录条目，由于 MDT 处于不活跃状态，使用 xmqit 将导致 IO 错误。请注意，如果 MDT 可用，则应使用标准的 rm -z 命令来删除远程目录。该删除操作完成后，管理员应使用以下命令将 MDT 标记为永久停用状态：letl conf param {MDT name}.mdc.active=0 [KG] unknown_source。3. 在特定的情况下或运行特定的命令，OST 可能会永久地在文件系统中停用。注意永久停用的 MDT 或 OST 仍会出现在文件系统配置中，直到使用 writeconf 重新生成配置或新 MDT 或 OST 在同一索引位置替代原设备并永久激活 [KG] unknown_source。4. 当 OST 文件系统完全无法访问时，OST 配置文件未备份时，即使 OST 文件系统完全无法访问，仍可在相同索引处用新的 OST 替换故障 OST。请在格式化时使用 --replace 选项：oss# mkfs.lustre --ost --reformat --replace --index=old_ost index \\\\other options /dev/new_ost dev [KG] unknown_source。5. 挂载 OST 文件系统，oss# mkdir /mnt/ost oss# mount -t ldiskfs /dev/new_ost dev /mnt/ost。
